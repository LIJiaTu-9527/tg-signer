# WebUI 使用文档

本文档介绍如何使用 TG Signer 的 WebUI 页面完成账号登录、任务配置和日常运维。

## 启动方式

本地：

```bash
tg-signer webgui -H 127.0.0.1 -P 8080
```

服务器：

```bash
tg-signer webgui -H 0.0.0.0 -P 8080 --auth-code your-access-code
```

如果你使用 Docker，参考 [Docker 部署文档](../DOCKER_DEPLOY.md)。

## 页面结构

### 1. 登录

用途：

- 选择账号
- 设置 session 目录
- 设置代理
- 发送验证码
- 提交验证码
- 提交二步验证密码
- 退出登录

建议：

- 一个 Telegram 账号对应一个账号名，例如 `main_account`
- 同一服务器多账号时，统一放在同一个 `session_dir` 下管理

### 2. 运行配置

用途：

- 选择运行账号
- 指定该账号需要保活运行哪些 signer 配置
- 指定该账号需要保活运行哪些 monitor 配置
- 保存保活运行配置
- 只启动 signer
- 只启动 monitor
- 启动全部选中配置
- 查看后台任务状态
- 单独停止或删除某个后台任务

说明：

- signer 和 monitor 可以分开运行
- 可以只有 signer，没有 monitor
- 也可以只有 monitor，没有 signer
- 页面会显示当前账号正在运行的 signer / monitor 配置

### 3. 即时操作

用途：

- 选择使用哪个账号执行即时操作
- 发送文本消息
- 发送 Dice
- 查询群成员 / 管理员
- 创建 Telegram 定时消息
- 查看已配置的定时消息

注意：

- 当前页面发送消息时，必须先选账号
- 如果报 `PEER_ID_INVALID`，通常是这个账号还没有见过该目标会话

最稳的测试方式：

- 账号：选择已登录账号
- 目标：`me`
- 文本：任意内容

### 4. LLM 配置

用途：

- 配置 OpenAI 兼容接口的 API Key、Base URL、Model

说明：

- 该页面是可选的
- 不配置也可以使用普通签到、监控和即时消息功能

### 5. 配置

用途：

- 管理 signer 配置
- 管理 monitor 配置
- 导入 / 导出 JSON

### 6. 用户

用途：

- 查看缓存的用户资料
- 查看 recent chats

### 7. 记录

用途：

- 查看各个签到任务的 `sign_record.json`

### 8. 日志

用途：

- 查看文件日志
- 查看 WebUI 捕获到的运行日志

## 典型操作流程

### 场景一：首次使用

1. 进入 `登录`
2. 选择账号名并完成 Telegram 登录
3. 进入 `配置` 创建 signer 或 monitor 配置
4. 进入 `运行配置` 绑定账号与配置
5. 启动保活任务

### 场景二：测试即时发消息

1. 进入 `即时操作`
2. 选择发送账号
3. 目标填 `me`
4. 文本填 `web test`
5. 点击 `发送文本`

### 场景三：查看任务是否真的在跑

1. 打开 `运行配置`
2. 查看“当前账号正在运行的配置”
3. 查看“后台任务”卡片中的状态
4. 需要清理历史记录时，点击单条任务后的 `删除`

## 常见问题

### 任务删除后为什么上面显示无，下面还有记录

上面的“当前账号正在运行的配置”只反映正在运行的任务。

下面“后台任务”包含运行历史：

- `running`：正在运行
- `completed`：正常结束
- `cancelled`：手动停止
- `failed`：执行失败

如果只想删除某一条历史，直接点击该任务卡片上的 `删除`。

### LLM 没配能不能用

可以。只有 AI 相关动作不可用，普通功能正常。

### 登录成功但即时消息发不出去

优先检查：

- 页面里选的是不是正确账号
- 目标是不是 `me`
- 目标会话是否被该账号“见过”

如果发 `me` 成功，说明 WebUI 本身没有问题。
